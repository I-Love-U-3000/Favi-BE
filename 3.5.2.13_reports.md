# 3.5.2.13. Bảng dữ liệu: REPORTS

## 1. Mô tả
Bảng lưu trữ các vé báo cáo vi phạm (Ticket) do người dùng gửi đến ban quản trị.

## 2. Chi tiết thuộc tính

| STT | Attribute Name | Data Type | Constraint | Meaning or Note |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Id | uuid | PK, NOT NULL | Khóa chính |
| 2 | ReporterId | uuid | FK (PROFILES), NOT NULL | ID người báo cáo |
| 3 | TargetType | integer | NOT NULL | Loại đối tượng bị báo cáo (0: Post, 1: Comment, 2: User...) |
| 4 | TargetId | uuid | NOT NULL | ID của đối tượng bị báo cáo (Polymorphic ID) |
| 5 | Reason | text | NULL | Lý do báo cáo chi tiết |
| 6 | Status | integer | NOT NULL, DEFAULT 0 | Trạng thái xử lý (0: Pending, 1: Resolved, 2: Rejected) |
| 7 | Data | json | NULL | Dữ liệu snapshot của đối tượng tại thời điểm báo cáo |
| 8 | CreatedAt | timestamp | NOT NULL, DEFAULT NOW() | Thời điểm gửi báo cáo |
| 9 | ActedAt | timestamp | NULL | Thời điểm Admin xử lý xong |
