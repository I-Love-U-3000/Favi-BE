# Use Case Descriptions

Based on the Use Case Diagrams, here are the definitions for the system use cases.

| # | Use Case Name | Definition |
| :--- | :--- | :--- |
| 1 | Sign In / Sign Up | User authenticates into the system using credentials or registers a new account to access platform features. |
| 2 | Manage Profile | User views and updates their personal information, avatar, bio, and privacy settings. |
| 3 | View Newsfeed & Explore | User views a personalized feed of posts from friends and explores trending content suggested by algorithms. |
| 4 | Create & Manage Posts | User creates new posts with multimedia (images, videos), updates existing posts, or deletes them. |
| 5 | Interact (Like, Comment, Share) | User engages with content by reacting (liking), adding comments, or sharing posts to their own feed or others. |
| 6 | Chat (1-on-1, Group) | User sends and receives real-time messages in private conversations or group chats. |
| 7 | Manage Collections | User organizes saved posts into private or public collections for easy access. |
| 8 | Friend & Follow | User manages connections by sending friend requests, accepting/denying requests, or following other users. |
| 9 | Manage Notifications | User views system notifications and configures preferences for alerts. |
| 10 | Create & Join Groups | User creates new community groups or joins existing ones to interact with members sharing similar interests. |
| 11 | Report Content | User flags inappropriate posts, comments, or users for administrative review. |
| 12 | Switch to Pro Mode | User upgrades their account to a professional profile to access creator tools. |
| 13 | View Professional Dashboard | Creator views analytics and insights regarding their content performance and audience engagement. |
| 14 | Manage Ad Campaigns | Creator creates and manages paid advertisement campaigns to boost post visibility. |
| 15 | Sign In (Admin) | Administrator logs into the secure management portal. |
| 16 | Moderate Users & Content | Admin reviews flagged content/users and takes action (ban user, remove content, warn). |
| 17 | View Reports (Admin) | Admin views a list of reported items and investigates the details of each report. |
| 18 | Monitor System Health | Admin views the status of system services, identifying potential outages or performance issues. |
| 19 | View Application Logs | Admin accesses detailed system logs for debugging and auditing purposes. |
